Testing Video4Linux device priorities

- nxtvepg relies on driver
- explanation bg acq: always-on
- principle
- howto

This test application allows to check if your video card driver supports
the handling of "device priorities", which allow background VBI applications
to play nicely together with interactive applications such as TV apps,
or video recording.  For the technical background see
http://nxtvepg.sourceforge.net/v4l2/

For the test unpack the tarball in an empty directory, adapt the Tcl/Tk
paths in the Makefile if necessary (same as when compiling nxtvepg) and
compile with "make", then start ./sprio-test and enter the path to your
video card device at the top.

If the device doesn't support the priority ioctl, you'll immediately
get an error message in the text field at the top.  If not, check the
"current device priority level": should be "background" by default,
or "interactive" when a TV application is running.  You should get an
error message when pressing button "Test" while a TV app is running,
unless you increase the priority level up to the current device priority
level.

You can also test the driver without starting a TV app by using the
two different priority and frequency control sections.  When assigning
different priorities in the two sections, changing channels should fail
for the section with the lower priority.

If you find your driver doesn't support priorities, please upgrade to
it's latest version.  If it still doesn't work, please complain to the
device author and/or send me a mail.
