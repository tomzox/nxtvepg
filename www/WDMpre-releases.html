WDM Support for nxtvepg:
alpha testers download page

This page lists the latest updates for the very first attempt to support Windows WDM devices for nxtvepg software package. The software which is available here is only intended for developers and alpha/beta testers. If you're searching for the regular release visit the download page.

Note: currently there are two development branches: besides WDM support there's also a 2.7.0 maintanance branch. The two branches will be merged for release 2.8.0.

Introduction

The following pre-release contains the latest Windows release of nxtvepg with WDM support, i.e. the option to use drivers supplied by your card's manufacturer instead of the one which comes with nxtvepg. Main goal of development is to help Windows users who cannot use nxtvepg at all because their TV card is not supported by nxtvepg's driver. This is because currently VBI acquisition in the Windows port of nxtvepg is based exclusively on the DScaler "dsdrv4" driver, which supports only cards based on Conexant BT878, CX23881 or Philips SAA7134 (and even among these only the ones with supported tuners.) Cards with chip sets from vendors which do not disclose specifications to support open source driver implementations cannot be supported this way.

WDM support is useful for Windows users only. UNIX users need not look further.

Please do also refer to the mailing list archives and forum for background information.

Known problems

The following bugs/open issues are already known:

  • When initially selecting a WDM source in the TV card input configuration dialog, video source selection may not work (device is allegedly busy.) If that happens, just close the dialog with OK and open it again. After that the video input selection should work.
  • TV channel/frequency conversion doesn't work for all channel tables yet. If you live in Germany and have cable, you need to create a file named VBIAcqWDMDrv.ini in the same directory as VBIAcqWDMDrv.dll with the following two lines: 
    [Debug Behaviour]
ForceRFInput=c
    This file is also in WDMDebugPkgDDMMYY.zip (see download section below), including documentation and additional debug options.
  • The EPG provider scan must be run in slow mode (you can enable this option at the bottom of the provider scan dialog window.) Even then some networks may appear duplicate in the scan. If your provider isn't found due to this, try running the scan again, or use a K!TV or DScaler channel table and move your provider's channel to the first position.
  • Hauppauge PVR 250/350 cards are not supported yet.

Other problems which were formerly listed here (i.e. "elaborate VBI slicer" not implemented and dsdrv driver disabled) are solved now.

Tests feedback

This section gives cards and system combination currently reported as "working".

  • Leadtek WinFast TV2000XP Expert on Windows XP and Windows 2000
  • Hauppauge WinTv PCI (Bt878 + Philips Tuner) on Windows XP
  • Hauppauge WinTv Go on Windows XP
  • Hauppauge WinTV PVR on Windows 2000
  • Hauppauge WinTV PCI (new series with CX23881 chip) on W2K: works partly
    Appearently due to a bug in the WDM driver apx. half of the teletext pages are lost; but if you're patient EPG reception is still possible.
  • Typhoon card "Art.No. 90031" with Philips capture chip (SAA7134) on W2K
  • Almost any card with Brooktree Bt878 or Bt848 capture chip when using the free btwincap WDM driver.

If you want to provide feedback, please send it to Gérard, not to Tom (for developers' mail addresses look in the SourceForge project member list) Please ask first if your card is already tested before sending large trace files.

Change history

API Synchronization with 2.8.0pre3 and up and bug fix (2005-03-31)

  • There where an API change between nxtvepg and the WDM driver DLL that needed fix.
  • In the future, there will be a DLL version check to prevent use of wrong DLL.
  • Fixed a bug that caused VBI data loss.
  • DirectX ressources management bug fix : enable / disable acquisition cycles now works.

Merged WDM support into 2.8.0pre1 (2005-01-20)

  • From now, there is no more need to patch nxtvepg.exe as WDM support is integrated into.

Merged WDM support into 2.7.5pre1 (2005-01-14)

  • Added lot of debugging facilities.
  • Some more TV cards reported working.

Start of WDM alpha support on the 2.7.0 (2004-04-30)

  • Added WDM Support.
  • First time of issue. Only tested with Leadtek WinFast TV2000XP Expert and Windows XP.
