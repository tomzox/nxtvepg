WDM Support for nxtvepg:
alpha testers download page

This page lists the latest updates for the very first attempt to support Windows WDM devices for nxtvepg software package. The software which is available here is only intended for developers and alpha/beta testers. If you're searching for the regular release visit the download page.

Note: currently there are two development branches: besides WDM support there's also a 2.7.0 maintanance branch. The two branches will be merged for release 2.8.0.

Introduction

The following pre-release contains the latest Windows release of nxtvepg with WDM support, i.e. the option to use drivers supplied by your card's manufacturer instead of the one which comes with nxtvepg. Main goal of development is to help Windows users who cannot use nxtvepg at all because their TV card is not supported by nxtvepg's driver. This is because currently VBI acquisition in the Windows port of nxtvepg is based exclusively on the DScaler "dsdrv4" driver, which supports only cards based on Conexant BT878, CX23881 or Philips SAA7134 (and even among these only the ones with supported tuners.) Cards with chip sets from vendors which do not disclose specifications to support open source driver implementations cannot be supported this way.

WDM support is useful for Windows users only. UNIX users need not look further.

Please do also refer to the mailing list archives and forum for background information.

Known problems

The following bugs/open issues are already known:

  • When initially selecting a WDM source in the TV card input configuration dialog, video source selection does not work (device is allegedly busy.) Just close the dialog with OK and open it again. After that the selection should work.
  • The "elaborate" VBI slicer is not implemented yet. You should select slicer quality "simple" in the TV card input configuration dialog.
  • Hauppauge PVR 250/350 cards are not supported yet.
  • The dsdrv driver is (intentionally) disabled in this release, i.e. only WDM sources are currently enabled in this branch.

Tests feedback

This section gives cards and system combination currently reported as "working".

  • Leadtek WinFast TV2000XP Expert on Windows XP and Windows 2000
  • Hauppauge WinTv PCI (Bt878 + Philips Tuner) on Windows XP

If you want to provide feedback, please send it to Gérard, not to Tom (for developers' mail addresses look in the SourceForge project member list) Please ask first if your card is already tested before sending large trace files.
