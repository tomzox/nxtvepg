Download

Die letzte stabile Distribution ist Version 2.6.0 vom 11. Oktober 2003. Sie enthält mehrere neue Features und Verbesserungen, siehe Change-Log 2.6.0. Ältere Distributionen sind über die SourceForge File List verfügbar.

Source Distribution

nxtvepg-2.6.0.tar.gz

Neueste stabile Source Distribution für Linux, NetBSD und FreeBSD: entpacken Sie das Archiv, wechseln Sie in das angelegte Verzeichnis mit einer Shell und geben Sie die Kommandos "make" und "make install" ein. Detailliertere Infos können Sie der README Datei entnehmen. Die Windows Version kann natürlich auch aus den Sourcen compiliert werden; eine Anleitung dazu findet sich auch im README.

Aus dem source tarball läßt sich auch ein RPM package bauen mit dem Kommando: rpm -ta nxtvepg-2.6.0.tar.gz. Das erzeugte package kann dann wie üblich mit rpm -i installiert werden. Alternativ sind auch fertige packages verfügbar (jedoch nicht unbedingt immer die neueste Version):

  • Für Linux von rpmfind.net oder Debian
  • Für NetBSD von netbsd.org
  • Für FreeBSD von freebsd.org

Binärdistribution für M$ Windows

nxtvepg-2.6.0-win.zip

Windows executable der letzten stabilen Version: Das nxtvepg ZIP Archiv muß lediglich in ein leeres Verzeichnis entpackt werden, dann kann nxtvepg.exe direkt gestartet werden. Ihr INI file und die Datenbanken von älteren nxtvepg Installationen können Sie ins neue Verzeichnis übernehmen. Das tvsim ZIP file kann optional danach in dasselbe Verzeichnis entpackt werden. Eine Beschreibung der initialen Konfiguration findet sich im Hilfe Menü bzw. in der HTML Benutzeranleitung, Abschnitt Getting Started (nur in englisch verfügbar).

Für Entwickler: Demo zur Interaktion mit TV Applikationen (siehe auch help wanted): nxtvepg-2.6.0-tvsim.zip (siehe auch tvsim, vbirec oder vbiplay manuals, oder tvsim & nxtvepg und tvsim & vbirec screenshot)

Demo Datenbank

nxdemodb.epg.gz

Falls Sie keine der oben genannten nexTView Programmanbieter empfangen, können Sie die Software dennoch im Demo-Modus ausprobieren. Dazu können Sie die Demo-Datenbank runterladen, dekomprimieren und dann mit den Kommandozeilenargumenten -demo nxdemodb.epg laden oder im Windows-Explorer einfach per Drag-and-Drop auf nxtvepg.exe ziehen.

Das Urheberrecht am Inhalt dieser Datenbank liegt bei Swiss Tele text, deshalb bitte nicht weiterverbreiten.


Systemanforderungen

Minimale Hardwarevoraussetzungen sind ein PC mit einer Performance enstprechend einer 90 MHz Intel Pentium CPU und 64 MB RAM (nxtvepg läuft natürlich auch auf AMD und sogar auf 64-Bit Plattformen wie Power-PC und Ultra-Sparc.) Ebenfalls benötigt wird eine TV-Karte (analog, d.h. kein DVB.)

Die Windows Version unterstützt alle PCI Karten mit den Capture Chips Brooktree Fusion Bt878A / Bt848, Nachfolger Conexant CX23881 oder Philips SAA7134. Bitte beachten daß derzeit keine AGP Karten oder USB Boxen unterstützt werden.

Die Linux Version unterstützt alle Karten für die es video4linux oder v4l2 kompatible Treiber gibt (z.B. bttv, saa7134 oder cx8800 oder USB TV Empfänger mit VBI support.) Die NetBSD/FreeBSD Version unterstützt alle Karten die vom bktr Treiber unterstützt werden.

Linux Softwareanforderungen

  • kernel >= 2.0.35 (upgrade auf 2.2.x für bttv-Nutzer wärmstens empfohlen)
  • bttv (version >= 0.7.35 empfohlen) oder saa7134 Treiber und video4linux
  • GNU C Compiler (falls Sie nxtvepg nicht in Form eines RPM Binärarchivs herunterladen)
  • Tcl/Tk Version 8.3.2 oder neuere (8.4.4 empfohlen) verfügbar unter http://www.tcl.tk/

NetBSD Softwareanforderungen

  • Kernel >= 1.5 mit dem bktr Treiber Version 2.17 oder neuer,
  • GNU C Compiler (falls Sie nxtvepg nicht in Form eines vorkompilierten NetBSD packages herunterladen)
  • Tcl/Tk Version 8.3.2 oder neuere (8.4.4 empfohlen) verfügbar unter http://www.tcl.tk/

FreeBSD Softwareanforderungen

  • FreeBSD 4-STABLE ab dem 1. November 2000 oder FreeBSD 5, d.h. ein System, das den bktr(4)-Treiber in der Version 2.18 oder neuer enthält. Ältere FreeBSD Installationen (2.x, 3.x, 4.x) können mit den neuesten bktr Treiber aktualisiert werden.
  • Tcl/Tk Version 8.3.2 oder neuere (8.4.4 empfohlen).
    Installationshinweise:

    Aktualisieren Sie Ihre Ports-Collection und installieren Sie den ports/multimedia/nxtvepg Port.

    Alternativ können Sie auch ein vorkompiliertes Packet mit dem folgenden Kommando pkg_add -r nxtvepg direkt vom FreeBSD FTP Server installieren.

    Falls Sie bei einen dieser Schritte Hilfe brauchen, dann sollte das FreeBSD Handbuch weiterhelfen

Windows Softwareanforderungen

  • nxtvepg benötigt keine zusätzliche Software, d.h. alle für den Decoder benötigte Software ist in der Distribution enthalten.
  • Ausnahme Windows-95: es wird die winsock2 DLL benötigt (ws2_32.dll, für das daemon feature), die im Gegensatz zu neueren Windows Versionen noch nicht in der Windows 95 Basisinstallation enthalten war.
  • Es werden alle Windows Versionen ab Windows 95 unterstützt.
  • Auf Windows-NT und Windows 2000 ist ein priviligierter Account erforderlich (z.B. Administrator) um den Treiber für die TV Karte zu installieren und starten.
