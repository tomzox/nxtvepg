Nxtvepg Download

Der neueste offizielle Release von nxtvepg ist Version 2.8.0, die letzte stabile Version ist 2.7.7. Der Unterschied zwischen den beiden sind eine Reihe neuer Features (siehe auch detaillierte Liste) im neuen Release 2.8.0. Falls Sie bereits eine funktionierende nxtvepg Installation haben, brauchen sie nur auf die neue Version umzustellen falls Sie an den Features Interesse haben. Für neue Anwender wird aber empfohlen gleich die neueste Version zu verwenden und nur bei Problemen auf eine ältere Version zurückzugreifen. (Bitte denken Sie daran Fehler im Forum zu melden damit sie behoben werden können.)

Falls Sie eine ältere Version von nxtvepg ersetzen, sollten Sie eine Kopie Ihrer alten nxtvepg.ini Datei aufheben damit Sie zur alten Version zurückkehren können. (Denn ältere Versionen von nxtvepg können INI Dateien die von neueren Versionen modifiziert wurden nicht mehr einlesen.) Beachten Sie außerdem daß bei Verwendung des nxtvepg daemons zusammen mit dem nxtvepg GUI beide Programme dieselbe Version haben müssen.

Ältere Distributionen sind jederzeit noch über die SourceForge File List verfügbar.

Source Distribution

  • nxtvepg-2.8.0.tar.gz
  • nxtvepg-2.7.7.tar.gz

Source Distribution für Linux, NetBSD und FreeBSD: entpacken Sie das Archiv, wechseln Sie in das angelegte Verzeichnis mit einer Shell und geben Sie die Kommandos "make" und "make install" ein. Detailliertere Infos können Sie der README Datei entnehmen. Die Windows Version kann natürlich auch aus den Sourcen compiliert werden; eine Anleitung dazu findet sich ebenso in der README Datei.

Aus dem source tarball läßt sich auch ein RPM package bauen mit dem Kommando: rpm -ta nxtvepg-2.8.0.tar.gz. Das erzeugte package kann dann wie üblich mit rpm -i installiert werden. Alternativ sind auch fertige packages verfügbar (jedoch nicht unbedingt immer die neueste Version):

  • Für Linux Red Hat/Fedora Core von ATrpms.net,
    für Debian von debian.org,
    für alle anderen Linux Distributionen rpmfind.net
  • Für NetBSD von netbsd.org
  • Für FreeBSD von freebsd.org

Binärdistribution für M$ Windows

  • nxtvepg-2.8.0-win.zip
  • nxtvepg-2.7.7-win.exe

Falls Ihre TV-Karte nicht vom nxtvepg-internen dsdrv4 Treiber unterstützt wird, können Sie eine separate WDM interface library herunterladen die es erlaubt den WDM Treiber des Kartenherstellers zu verwenden.

Wichtig für Benutzer von K!TV: Um nxtvepg 2.8.0 zusammen mit K!TV zu verwenden müssen Sie unbedingt die NextViewPlugin.dll library aus dem 2.8.0 Release in das Plugin Unterverzeichnis Ihrer K!TV Installation kopieren.

Zuguterletzt: Es gibt einer alternative Version von release 2.8.0: nxtvepg-2.8.0-win-utf8.zip. die intern UTF-8 die Zeichenkodierung verwendet mit XMLTV Dateien von außerhalb Westeuropas verarbeiten zu können.

Ein Windows-Installer für Version 2.8.0 wird in wenigen Tagen vefügbar sein. Bis dahin können sie die Software mittels der ZIP Datei installieren: Das nxtvepg ZIP Archiv muß lediglich in ein neues, leeres Verzeichnis entpackt werden, dann kann nxtvepg.exe direkt gestartet werden.

Ihr INI file und die Datenbanken von älteren nxtvepg Installationen können Sie ins neue Verzeichnis übernehmen. Eine Beschreibung der initialen Konfiguration findet sich im Hilfe Menü bzw. in der HTML Benutzeranleitung, Abschnitt Getting Started.

Für Entwickler: Demo zur Interaktion mit TV Applikationen (siehe auch FAQ): nxtvepg-2.8.0-tvsim.zip (siehe auch tvsim, vbirec oder vbiplay manuals, oder tvsim & nxtvepg und tvsim & vbirec screenshot)

Demo Datenbank

nxdemodb.epg.gz

Falls Sie keine der oben genannten nexTView Programmanbieter empfangen, können Sie die Software dennoch im Demo-Modus ausprobieren. Dazu können Sie die Demo-Datenbank runterladen, dekomprimieren und dann mit den Kommandozeilenargumenten -demo nxdemodb.epg laden oder im Windows-Explorer einfach per Drag-and-Drop auf nxtvepg.exe ziehen.

Das Urheberrecht am Inhalt dieser Datenbank liegt bei Swiss Tele text, deshalb bitte nicht weiterverbreiten.


Systemanforderungen

Minimale Hardwarevoraussetzungen sind ein PC mit einer Performance enstprechend einer 90 MHz Intel Pentium CPU und 128 MB RAM (nxtvepg läuft natürlich auch auf AMD und sogar auf 64-Bit Plattformen wie Power-PC und Ultra-Sparc.) Ebenfalls benötigt wird eine TV-Karte (analog, d.h. kein DVB.)

Die Windows Version unterstützt mittels des mitgelieferten "dsdrv4" (DScaler) Treibers die meisten aller PCI Karten mit den Capture Chips Brooktree Fusion Bt878A / Bt848, Nachfolger Conexant CX23881 oder Philips SAA7134. Viele weitere TV-Karten werden mittels einer separaten WDM Interface DLL unterstützt.

Die Linux Version unterstützt alle Karten für die es video4linux oder v4l2 Treiber gibt (z.B. bttv, saa7134 oder cx8800 oder USB TV Empfänger mit VBI support.) Akquisition via DVB Karten ist (für fortgeschrittene Nutzer) möglich wenn nxtvepg selbst compiliert wird unter Verwendung der ZVBI library (dazu "-DUSE_LIBZVBI" im nxtvepg Makefile ent-kommentieren.) Die NetBSD/FreeBSD Version unterstützt alle Karten die vom bktr Treiber unterstützt werden.

Linux Softwareanforderungen

  • video4linux Treiber mit VBI-Unterstützung für Ihre TV-Karte. Ab Kernel Version 2.6.x sind video4linux Treiber für die gängisten analogen TV-Karten standardmäßig im Linux Kernel enthalten. Einschränkungen: bttv Treiber in Kernel Versionen 2.6.16 oder neuer werden erst ab nxtvepg 2.7.7 unterstützt. bttv Treiber in Kernel Versionen 2.6.21 oder neuer funktionieren erst ab nxtvepg 2.8.0.
    Ältere Kernel Versionen::
    • Für TV-Karten mit Brooktree Bt848 oder Bt878 Capture-Chip: Kernel 2.0.35 oder neuer und bttv Treiber Version >= 0.7.55; bei Kernel 2.4 und älter wurde zudem ein separates video4linux Modul benötigt.
    • Für TV-Karten mit Philips SAA7134 Capture-Chip: Kernel 2.4.x mit saa7134 Treiber 0.2.9 oder neuer plus video4linux Modul.
    • Für TV-Karten mit Conexant 2388x Capture-Chip: Kernel 2.4.x mit cx88xx Treiber aus snapshot video4linux-20040330 oder neuer (offizielle cx8800 Treiber-Releases gibt es erst seit bzw. innerhalb von Kernel 2.6.x)
  • GNU C Compiler Version 2.95.3 oder 3.2 oder neuer, falls Sie nxtvepg aus den Sourcen compilieren (d.h. falls Sie nxtvepg nicht fertig compiliert über einen Paketmanager herunterladen.)
  • Tcl/Tk Version 8.4 oder neuer (Version 8.5 empfohlen wegen des modernisiertes Aussehens.)
  • X11 Bibliotheken Version X11R5 oder neuer (Xfree86 oder Xorg Releases.)

Anmerkung: Die Tcl/Tk und X11 Bibliotheken werden nur für das GUI benötigt. Bei alleiniger Verwendung des "nxtvepgd" Akquisitionsprozesses entfällt die Voraussetzung.

NetBSD Softwareanforderungen

  • Kernel >= 1.5 mit dem bktr Treiber Version 2.17 oder neuer,
  • GNU C Compiler (falls Sie nxtvepg nicht in Form eines vorkompilierten NetBSD packages herunterladen)
  • Tcl/Tk Version 8.4 oder neuer (Version 8.5 empfohlen wegen des modernisiertes Aussehens.)
  • Für das GUI wird selbstverständlich außerdem die X11 Bibliotheken und Server benötigt. Bei alleiniger Verwendung des "nxtvepgd" Akquisitionsprozesses entfällt diese Vorraussetzung.

FreeBSD Softwareanforderungen

  • FreeBSD 4-STABLE ab dem 1. November 2000 oder FreeBSD 5, d.h. ein System, das den bktr(4)-Treiber in der Version 2.18 oder neuer enthält. Ältere FreeBSD Installationen (2.x, 3.x, 4.x) können mit den neuesten bktr Treiber aktualisiert werden.
  • Tcl/Tk Version 8.4 oder neuer (Version 8.5 empfohlen wegen des modernisiertes Aussehens.)

Installationshinweise: Aktualisieren Sie Ihre Ports-Collection und installieren Sie den ports/multimedia/nxtvepg Port. Alternativ können Sie auch ein vorkompiliertes Packet mit dem folgenden Kommando pkg_add -r nxtvepg direkt vom FreeBSD FTP Server installieren. Falls Sie bei einem dieser Schritte Hilfe brauchen, dann sollte das FreeBSD Handbuch weiterhelfen.

Windows Softwareanforderungen

  • nxtvepg benötigt keine zusätzliche Software, d.h. alle für den Decoder benötigte Software ist in der Distribution enthalten.
  • Ausnahme Windows-95: es wird die winsock2 DLL benötigt (ws2_32.dll, für das daemon feature), die im Gegensatz zu neueren Windows Versionen noch nicht in der Windows 95 Basisinstallation enthalten war.
  • Es werden alle Windows Versionen ab Windows 95 unterstützt.
  • Auf Windows-NT und Windows 2000 ist ein priviligierter Account erforderlich (z.B. Administrator) um den Treiber für die TV Karte zu installieren und starten.
