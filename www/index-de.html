nxtvepg
Browser für TV-Programmlisten
im XMLTV Format

Herzlich willkommen auf der Homepage von nxtvepg - einer Software zur Darstellung von TV-Programminformationen ("Elektronischer Programmführer" aka EPG) die im XMLTV Format gespeichert sind. nxtvepg ermöglicht ein flexibles Zusammenführen von Programminformationen aus verschiedenen Quellen und deren Anzeige in Form einer Liste oder in Form einer Tabelle ähnlich TV-Programmzeitschriften auf Papier. nxtvepg erlaubt dabei Eingrenzung (aka "Filterung") der angezeigten Programme anhand einer Vielzahl von Kriterien, die Speicherung solcher Eingrenzung als "Shortcuts" und deren Verwendung zur Markierung ausgewählter Programme in der Übersicht mittels selbstdefinierter Spalten. Erinnerungen können für einzelne Sendungen als auch basierend auf Shortcuts programiert werden.

nxtvepg enthält außerdem einen "Teletext EPG Grabber", der mittels einer TV-Karte Programminformationen aus der TV-Programmübersicht im Teletext ausgewählter deutscher TV-Sender extrahieren kann. Die so erzeugten Programmlisten können direkt in nxtvepg angezeigt werden, oder für Verwendung in anderen Programmen in Dateien im XMLTV Format exportiert werden.

Historie: In den Jahren 2000-2008 wurde nxtvepg ursprünlich entwickelt zur Dekodierung und Anzeige von Nextview EPG. Dies war ein Europäischer Standard zur Übertragung von Elektronischen Programmzeitschriften im analogen TV-Signal. Durch den weltweiten Wechsel auf digitale Übertragungsstandards wie DVB ist dieser Standard jedoch obsolet und wurde daher im nxtvepg release 3.0 entfernt. Mehr dazu....

Aktueller release 3.1 (23. Juli 2023)

Ziel dieses Releases sind Verbesserungen der Unterstützung für den XMLTV Standard und damit der im nxtvepg GUI darstellbaren EPG Informationen. Zusätzlich wurden eine Reihe von alten Fehlern behoben.

  • Die Verarbeitung von Themenkategorien wurde von einer vordefinierten Liste umgestellt auf eine Liste die dynamisch aus den in der importieren XMLTV Datei tatsächnlich vorhandenen Kategoriebezeichnern generiert wird. Ein neuer Dialog im GUI erlaubt Kategorien auszuwählen und als Filter anzuwenden.
  • Mehrere neue Informationselemente des XMLTV Standards werden unterstützt: "review", "new", "premiere" und "last-chance". Unterstützung für "audio", "video" und "subtitles" wurde aktualisiert um neuere Attribute anzeigen zu können. (Von diesen Verbesserungen profitieren jedoch nur Nutzer die eine XMLTV Quelle verwenden die diese Informationselemente auch anbietet.)
  • Darstellung der "Editorial Ratings" (Filmbewertungen) wurde verbessert indem die Information genauso dargestellt wird wie im XMLTV enthalten (d.h. als "X von Y Sterne"). Außerdem werden zwei neue Alterseinstufungssysteme für Polen bzw. USA unterstützt und Rundung entfernt.
  • Der "Expire time" Filter, der die Anzeige bereits abgelaufener Sendungen kontrolliert, wurde um eine Einstellung erweitert die es erlaubt, ihn komplett abzuschalten, so dass auch sehr alte EPG Datenbanken noch komplett betrachtet werden können.
  • Fehler in der Textsuche wurden korrigiert, so dass diese nun auch wieder funktioniert wenn Unterscheidung von Groß-/Kleinschreibung abgeschaltet ist. (Der Fehler bestand seit der internen Umstellung auf die UTF-8 Textkodierung in Release 3.0.0).
  • Die Beschränkung auf 254 Sender beim XMLTV Import wurde auf ca. 8000 Sender angehoben und die Verarbeitung optimiert so dass sehr große Dateien in akzeptabler Zeit (d.h. wenigen Sekunden auf durchschnittlichen PCs) geladen werden können.
  • Die "Programmindex" und "Serien" Filter wurden ebenfalls für große Datenbanken optimiert.
  • Benutzerfreundlichkeit des "Load XMLTV file" Menübefehls wurde verbessert: Dieser Befehl öffnet nun einen normalen Dateiauswahldialog. Der bisherige, komplexere Dialog ist noch als "Browse XMLTV files" Befehl vorhanden mit einigen Verbesserungen in der Bedienbarkeit.
  • Benutzerfreundlichkeit der Sendernamenkonfiguration wurde verbessert durch Reduktion der Informationen und Auswhlmöglichkeiten auf solche die in der aktuellen Konfiguration relevant sind.
  • Auf MS Windows wird die "nxtvepg.ini" Konfigurationsdatei jetzt im "AppData" Verzeichnis im Benutzer-Home-Verzeichnis abgelegt, anstelle des aktuellen Arbeitsverzeichnisses. Dadurch wird der aktuelle Release Konfigurationsdateien von älteren Releases nicht automatisch laden.
  • Die Textenkodierung in der "nxtvepg.ini" Konfigurationsdatei wurde auf UTF-8 umgestellt um alle aus XMLTV stammendedn Texte unverändert speichern zu können. Auf Rechnern wo UTF-8 nicht Systemstandard ist, empfiehlt es sich, die alte Konfigurationsdatei zu löschen. ("nxtvepg.ini" auf MS Windows, bzw. "config/nxtvepg/nxtvepg.rc" auf Linux.)

Feedback

Feedback jeglicher Art ist jederzeit willkommen, sowohl in Form von Fragen, Verbesserungsvorschlägen, Feature requests oder bug reports. nxtvepg hat aktuell sehr wenige Nutzer, daher sollten Sie nicht darauf hoffen dass Bugs behoben werden ohne sie selber zu melden. Folgende Möglichkeiten stehen dafür zur Verfügung:

  • Liste der häufig gestellten Fragen (FAQ) (inklusive Antworten - Stand 2008)
  • Diskussionsforum: die einfachste Möglichkeit (erfordert keine Sourceforge Registrierung). Wenn möglich bitte in Englisch schreiben, oder ansonsten ein [de] dem Titel voranstellen.
  • Mailing list: Diese Mailing Liste wurde in der Vergangenheit hauptsächlich zur Ankündigung von Software-Releases verwendet. Aufgrund der geringen Nutzerzahl wird die Liste aktuell nicht mehr verwendet. Bitte verwenden Sie stattdessen das Forum.

Bevor Sie eine Frage stellen überlegen Sie bitte ob sie durch das manual geklärt wird und sehen Sie im Forum nach ob dieselbe Frage bereits beantwortet wurde.
