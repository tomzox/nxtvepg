nexTView
EPG Dekoder Software

Die hier zum Download erhältliche Software ermöglicht die Nutzung von nexTView - einer kostenlosen elektronischen Programmzeitschrift die zusammen mit dem Teletext (Videotext) im analogen TV-Signal übertragen wird. Mit dem Dekoder erhalten Sie das Fernsehprogramm aller überregionalen Sender in Deutschland, Österreich, der Schweiz und Frankreich. Einen ersten Eindruck können Sie sich mit diesem Linux screenshot oder diesem Windows screenshot verschaffen (alternativ neues Layout seit Version 2.5.0)

Der nexTView Standard wurde von den führenden TV Herstellern unter Führung des European Telecommunications Standards Institute (ETSI) von 1995-1997 entwickelt und ging erstmals zur Internationalen Funkausstellung in Berlin 1997 auf Sendung. Die Absicht war, eine Programmzeitschrift in den Fernseher zu integrieren. Die Übertragung basiert deshalb auf Teletext, jedoch wird ein kompaktes Binärformat anstelle von vorformatierten Textseiten verwendet. Der Vorteil gegenüber den normalen Papier-TV-Zeitschriften ist daß man sofort einen Überbick über alle gerade laufenden Sendungen erhält und zudem die Programmdatenbank nach einer Vielzahl Kriterien durchsuchen kann.

Der nxtvepg Dekoder erlaubt es nun, nexTView auch am PC zu nutzen und bietet gleichzeitig eine Menge neuer Features, die in einem TV-Gerät nicht realisierbar wären. Vorbedingungen für Sie als Nutzer sind daß Sie (a) eine TV-Tuner-Karte im PC stecken haben und (b) damit einen der Sender empfangen können auf dem nexTView übertragen wird.

Momentan wird nexTView auf den folgenden Sendern ausgestrahlt:

  • in Deutschland und Österreich: Kabel1, 3Sat, RTL-II, EuroNews (Abdeckung: ca. 31 Sender)
  • in der Schweiz: SF1, TSR1, TSI1, EuroNews, 3sat, Kabel1 (Abdeckung: ca. 37 Sender)
  • in Frankreich: Canal+, M6, EuroNews, TV5 (Abdeckung: ca. 8 Sender)
  • in der Türkei: TRT-1 (TRT-int, Abdeckung: 17 Sender)

Nährere Details finden sich in der Anbietertabelle. Sollten Sie keinen der genannten Sender empfangen, können Sie mit dieser Software mit Ausnahme des Demo-Modus leider nichts anfangen. Mehr Infos dazu und zu allen weiteren Sofware-Features in der nxtvepg Dokumentation (nur in englisch verfügbar).

Falls nxtvepg Ihr Interesse geweckt haben sollte und Sie nun an einem TV mit integriertem nexTView interessiert sind, schauen Sie doch auf den Seiten der Hersteller vorbei: Grundig, Loewe, Metz, Philips, Sony, Thomson oder Quelle Universum.

Systemanforderungen

Hardwarevoraussetzungen sind ein PC oder Workstation mit mindestens einer 90 Mhz Intel Pentium oder leistungsstärkeren CPU und eine TV-Karte (analog, d.h. kein DVB). Die Windows Version unterstützt alle Karten mit den Capture Chips Brooktree Fusion Bt878A / Bt848, Nachfolger Conexant CX23881 oder Philips SAA7134 (neu mit release 2.5.0) Die Linux Version unterstützt alle Karten für die es video4linux kompatible Treiber gibt (z.B. bttv und saa7134). Die NetBSD/FreeBSD Version unterstützt alle Karten die vom bktr Treiber unterstützt werden.

Linux Softwareanforderungen

  • kernel >= 2.0.35 (upgrade auf 2.2.x für bttv-Nutzer wärmstens empfohlen)
  • bttv (version >= 0.7.35 empfohlen) oder saa7134 Treiber und video4linux
  • GNU C Compiler (falls Sie nxtvepg nicht in Form eines RPM Binärarchivs herunterladen)
  • Tcl/Tk Version 8.0 oder neuere (8.3 empfohlen) verfügbar unter ftp://www.scriptics.com/pub/tcl/

NetBSD Softwareanforderungen

  • Kernel >= 1.5 mit dem bktr Treiber Version 2.17 oder neuer,
  • GNU C Compiler (falls Sie nxtvepg nicht in Form eines vorkompilierten NetBSD packages herunterladen)
  • Tcl/Tk Version 8.0 oder neuere (8.3 empfohlen) verfügbar unter ftp://www.scriptics.com/pub/tcl/

FreeBSD Softwareanforderungen

  • FreeBSD 4-STABLE ab dem 1. November 2000, d.h. ein System, das den bktr(4)-Treiber in der Version 2.18 oder neuer enthält. Ältere FreeBSD Installationen (2.x, 3.x, 4.x) können mit den neuesten bktr Treiber aktualisiert werden.
  • FreeBSD 5 ist derzeit ungetestet (Erfahrunsberichte bitte an Simon Barner).
  • Tcl/Tk Version 8.0 oder neuere (8.3 empfohlen).
    Installationshinweise:
    • Bei Benutzung des vorläufigen Ports, der auf seine Aufnahme in die FreeBSD Ports Collection wartet, oder des vorkompilierten Packages wird Tcl/Tk bei Bedarf automatisch mitinstalliert.
    • Alternativ kann Tcl/Tk durch den Port ports/x11-toolkits/tk83 (siehe entsprechender Handbuchabschnitt) oder mittels pkg_add -r tk-8.3.4 direkt vom FreeBSD FTP-Server installiert werden.

Windows Softwareanforderungen

  • nxtvepg benötigt keine zusätzliche Software, d.h. alle für den Decoder benötigte Software ist in der Distribution enthalten.
  • Ausnahme Windows-95: es wird die winsock2 DLL benötigt (ws2_32.dll, für das daemon feature), die im Gegensatz zu neueren Windows Versionen noch nicht in der Windows 95 Basisinstallation enthalten war.
  • Es werden alle Windows Versionen ab Windows 95 unterstützt.
  • Auf Windows-NT und Windows 2000 ist ein priviligierter Account erforderlich (z.B. Administrator) um den Treiber für die TV Karte zu installieren und starten.

Download

Die letzte stabile Distribution ist Version 2.5.2 vom 1. Juli 2003. Gegenüber 2.5.1 wurden einige kleinere Fehler behoben.

Mehr Details und alle früheren Änderungen sind im Change-Log 2.5.2 dokumentiert. Falls mit der Software Probleme auftauchen, bitte melden. Ältere Distributionen sind über die SourceForge File List verfügbar.

Source Distribution

Download der letzten stabilen Source Distribution für Linux, NetBSD und FreeBSD: nxtvepg-2.5.2.tar.gz

Aus dem source tarball läßt sich auch ein RPM package bauen mit dem Kommando: rpm -ta nxtvepg-2.5.2.tar.gz. Das erzeugte package kann dann wie üblich mit rpm -i installiert werden.

Alternativ sind auch fertige packages verfügbar (jedoch nicht notwendigerweise immer die neueste Version):

  • Für Linux von rpmfind.net oder Debian
  • Für NetBSD von netbsd.org
  • Für FreeBSD von leo.org (Port-Gerüst und Package)

Binärdistribution für M$ Windows

Windows executable der letzten stabilen Version: nxtvepg-2.5.2-win.zip

Für Entwickler: Demo zur Interaktion mit TV Applikationen: nxtvepg-2.5.2-tvsim.zip
(siehe auch tvsim, vbirec oder vbiplay manuals, oder tvsim & nxtvepg und tvsim & vbirec screenshot)

Das nxtvepg ZIP Archiv muß lediglich in ein leeres Verzeichnis entpackt werden, dann kann nxtvepg.exe direkt gestartet werden. Ihr INI file und die Datenbanken von älteren nxtvepg Installationen können Sie ins neue Verzeichnis übernehmen. Das tvsim ZIP file kann optional danach in dasselbe Verzeichnis entpackt werden. Eine Beschreibung der initialen Konfiguration findet sich im Hilfe Menü bzw. in der HTML Benutzeranleitung, Abschnitt Getting Started (nur in englisch verfügbar).

Demo Datenbank
