Software download

The latest stable release is version 2.7.7. See the overview on the homepage for a list of contents, or refer to the more detailed changelog 2.7.7.

For older release packages see the SourceForge File List.

Source distribution

nxtvepg-2.7.7.tar.gz

Source distribution for Linux, NetBSD and FreeBSD: unpack the archive, then change into the source directory and compile with "make", install with "make install". If you encounter problems see the README file for more detailed instructions. Of course it's also possible to compile the Windows port from these sources; this is also explained in README.

You can build a RPM package from the source tarball simply by running rpm -ta nxtvepg-2.7.7.tar.gz and install it as usual with rpm -i.

Alternatively, pre-compiled packages are provided at the following sites (not necessarily the latest version):

  • For Linux Red Hat/Fedora Core from ATrpms.net,
    for Debian from debian.org,
    for other Linux distributions try rpmfind.net
  • For NetBSD at netbsd.org
  • For FreeBSD at freebsd.org

Binary distribution for M$ Windows

nxtvepg-2.7.7-win.exe

If you're upgrading from an older version of nxtvepg, you can copy your previous INI file and databases into the new directory. The optional tvsim ZIP file should be unpacked into the same directory. Please refer to the Help menu or HTML manual page, section Getting Started for setup instructions.

For developers: demo application for TV application interaction (see also FAQ): nxtvepg-2.7.7-tvsim.zip (see tvsim, vbirec or vbiplay manuals, or tvsim & nxtvepg and tvsim & vbirec screenshot)

Demo Database

nxdemodb.epg.gz

If you do not receive any of the nexTView provider channels, download and browse the demo database to check out what you're missing.

To use the demo database, download it into your nxtvepg directory, decompress it and load it with the command line arguments -demo nxdemodb.epg (Unix) into nxtvepg, or simply drag and drop it onto the executable (Windows Explorer).

The copyright of the content of this database is by Swiss Tele text - please do not redistribute.


System Requirements

Hardware minimum requirements are a PC or workstation with performance equivalent to a 90 MHz Intel Pentium CPU and 64 MB of RAM (note nxtvepg is also known to work fine on Power-PC and Ultra-Sparc platforms.) Additionaly a TV tuner card (analog, i.e. not DVB) is required.

The Windows version supports all PCI capture cards based either on the capture chips Brooktree Fusion Bt878/Bt848, sucessor Conexant CX23881 or Philips SAA7134. Please note that no AGP cards or USB TV boxes are supported!

The Linux version supports all cards for which a video4linux or v4l2 compatible driver exists (e.g. bttv, saa7134, cx8800, possibly even USB TV boxes.) Acquisition from DVB devices does reportedly not work yet. The NetBSD and FreeBSD versions support all cards which are supported by the bktr driver.

Linux software requirements

  • For bttv kernel >= 2.0.35 (2.0.x was prone to freezes with bttv though, hence upgrade to at least 2.2.x is highly recommended)
  • SAA7134 and CX23881 cards require at least kernel 2.4.x plus some v4l2 kernel patches (your Linux distribution may already include those if it comes with saa7134 support), see bytesex.org; alternatively use kernel 2.6.2 or later, which needs not be patched.
  • Kernel must have video4linux support enabled (preferrably as modules to simplify upgrades) and include bttv (at least version >= 0.7.55 recommended) or saa7134 (at least version 0.2.9); cx8800 has no official releases yet, use the latest development snapshot.
  • GNU C Compiler (version 2.95.3 or 3.2 or later) to compile nxtvepg
  • Tcl/Tk version 8.3.2 or later, available from http://www.tcl.tk/

NetBSD software requirements

  • kernel >= 1.5 including the bktr driver 2.17 or later,
  • GNU C Compiler, unless you got nxtvepg in a precompiled NetBSD package
  • Tcl/Tk version 8.3.2 or later (8.4.4 recommended) available from http://www.tcl.tk/

FreeBSD software requirements

  • FreeBSD 4-STABLE as of November 1st 2000 (or more recent) or FreeBSD 5, i.e. a system that contains the bktr(4)-driver 2.18 or later. Older installations of FreeBSD (2.x, 3.x, 4.x) may be upgraded with the latest bktr driver.
  • Tcl/Tk version 8.3.2 or later (8.4.4 recommended).
  • Installation instructions:

    Update your ports collection and install the ports/multimedia/nxtvepg port.

    Alternatively, you can install a pre-compiled package with the following command pkg_add -r nxtvepg directly from FreeBSD FTP server.

    If you need help with one of these steps, please have a look at the FreeBSD handbook

Windows software requirements

  • No additional software is required: nxtvepg is shipped with all the required components.
  • Exception for Windows 95: the winsock2 DLL is required (ws2_32.dll, for the daemon feature) which - in contrary to newer Windows versions - was not included with this OS.
  • All Windows versions starting with Windows 95 are supported.
  • Note: On Windows NT and Windows 2000 you need to run nxtvepg under an account with special privileges (you must be allowed to install & start a service)
